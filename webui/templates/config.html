🔀 DHCP 工作模式

选择 PXE 服务器如何与现有 DHCP 服务器配合工作

💡 DHCP Proxy 模式说明
  • 主 DHCP 服务器（如路由器）正常分配 IP 地址
  • PXE 服务器只添加 PXE 引导选项（Option 66/67）
  • 客户端从路由器获取 IP，同时从 PXE 服务器引导
  • 需要配置路由器的 DHCP 选项指向本服务器
填写路由器或上级 DHCP 服务器的 IP 地址
⚠️ Standalone 模式说明
  • PXE 服务器将作为唯一的 DHCP 服务器
  • 需要 关闭路由器的 DHCP 服务器
  • 所有客户端将从本服务器获取 IP
  • 适合没有其他 DHCP 服务器的环境

📡 路由器配置 (DHCP Proxy 模式)

如果使用 DHCP Proxy 模式，需要在路由器上配置 DHCP 选项：

# OpenWrt 路由器配置示例
# 登录路由器后台，进入 网络 -> DHCP/DNS -> 高级设置

# 添加 DHCP 选项:
# Option 66: TFTP 服务器地址
option 66 '192.168.8.4'

# Option 67: 启动文件名
# BIOS: ipxe.pxe 或 undionly.kpxe
# UEFI: ipxe.efi 或 snp.efi
option 67 'ipxe.pxe'

# 或者在 /etc/config/dhcp 中添加:
config dnsmasq
    option 'dhcp_option' '66,192.168.8.4'
    option 'dhcp_option' '67,ipxe.pxe'
💡 提示

iVentoy 在 Proxy 模式下会嗅探 DHCP 流量并注入 PXE 选项，客户端无需特殊配置即可自动从本服务器引导。

❓ 应该选择哪个模式？

场景 推荐模式 说明
家庭/办公室网络 DHCP Proxy 不中断现有设备的网络连接
专用 PXE 网络 Standalone 只有 PXE 客户端的网络
无法修改路由器 Standalone 无法配置路由器 DHCP 选项
临时测试环境 DHCP Proxy 快速测试，无需改网络配置
